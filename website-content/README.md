# Website Content Index

Everything you need for your portfolio site. Drop these into your site builder, CMS, or static site generator.

## Pages

1. **[Hero](hero.md)** — Name, tagline, one-liner, certifications. Goes at the top of your landing page.
2. **[About Me](about.md)** — Full bio. Career arc, what sets you apart, what you're looking for.
3. **[Skills & Expertise](skills.md)** — 5 major skill areas with portfolio evidence and tech stack table.
4. **[Projects Showcase](projects-showcase.md)** — All 13 projects with GitHub links, tech stack, and "why it matters" impact statements.
5. **[Stats](stats.md)** — Quick-hit numbers for a sidebar or stats banner.

## How to Use

### If using a static site (Hugo, Jekyll, Astro, etc.)
Copy each `.md` file into your content directory. The formatting is standard markdown.

### If using a website builder (Squarespace, Wix, etc.)
Copy the text content and paste into your sections. The markdown formatting will transfer cleanly.

### If hand-coding HTML
The heading hierarchy is consistent — `#` = page title, `##` = section header, `###` = sub-section. Convert to `<h1>`, `<h2>`, `<h3>` respectively.

## Suggested Site Structure

```
/
├── Hero section (hero.md)
├── Stats bar (stats.md)
├── About section (about.md)
├── Skills section (skills.md)
├── Projects section (projects-showcase.md)
└── Contact / CTA
```

## Updates

When the 6 remaining projects are completed, `projects-showcase.md` and `stats.md` should be refreshed. The content is written so the "In Progress" projects slot in without restructuring.
